# daily_digest V1 — 브리프

> 역할: V1 매일 뉴스 다이제스트 봇이 무엇을 하고 무엇을 하지 않는지, 사용자가 무엇을 경험하는지, 어떤 규칙이 그 경험을 만드는지 명시한다. 구현 방법·데이터 스키마는 포함하지 않는다(Stage 4 requirements에서 다룸).
> 대상: Stage 2 Concept 검토, Stage 3 Tech Research 진입자, Stage 4 requirements 작성자, 사용자 일괄 검토자.

---

## 1. 배경

팜보스 그룹(3법인: 정다운 산지유통·팜보스 온라인 커머스·시경 프랜차이즈, 2024년 매출 약 130억) 임직원은 **AI 트렌드**와 **농산물 유통 환경 변화**를 모두 추적해야 한다. 김종만 총괄대표는 3법인 전략을, 정은주 이사는 온라인 사업·마케팅을, 장석중 이사는 산지·GS리테일 납품을 책임진다. 일반 직원은 매일의 업무를 회사 사업 영역과 연결된 정보 안에서 시작해야 한다.

현재는 직원이 개별적으로 여러 사이트(농민신문·식품음료신문·Anthropic Blog·Hacker News 등)를 순회하거나 외부 뉴스레터(TLDR AI·미라클레터 등)에 의지한다. 그 결과:

- **시간 손실**: 매일 30분~1시간이 흩어지는 검색에 쓰인다.
- **편차**: 직원마다 받는 정보가 다르고, 회의 시 같은 기반 위에서 시작하지 못한다.
- **회사 키워드 누락**: 일반 뉴스레터는 "청도 복숭아 시세", "GS리테일 1차 밴드 정책" 같은 회사 직결 키워드를 다루지 않는다.

**V1 다이제스트 봇은 이 세 문제를 동시에 해결한다.** 매일 KST 07:30, 전 직원 메일함에 "AI 트렌드 / 농산물 유통 / 팜보스 관심 키워드" 3카테고리의 큐레이션을 자동 발송한다. 결정은 사람이, 수집·정리·요약은 봇이 한다.

## 2. 사용자 경험

### 2-1. 발송 흐름

```
KST 07:30 (출근 전)
  ↓
직원 메일함에 다이제스트 도착
"[팜보스 트렌드 5/19(월)] AI·농산물 유통 오늘의 뉴스 8건"
  ↓
직원이 메일을 연다 (5분 분량)
  ↓
카테고리 헤더 3개를 빠르게 훑는다:
  • AI 트렌드        (5~10건)
  • 농산물·유통       (5~10건)
  • 팜보스 관심 키워드 (5~10건)
  ↓
관심 항목의 "한국어 요약 2 문장 이내"를 읽고
원문 링크가 필요한 것만 클릭해 원문으로 이동
  ↓
업무 시작
```

### 2-2. 본문 구조 — 직원이 보는 것 (Discovery 결론 #1 반영)

```
─────────────────────────────────────────
📰 [팜보스 트렌드] 5월 19일 (월) 오전 7:30 KST
─────────────────────────────────────────

오늘 8건의 새 뉴스가 있습니다.
(소스 12개 중 11개 정상 수집, 1개 실패: 농민신문)

━━ ① AI 트렌드 (3건) ━━━━━━━━━━━━━

1. Anthropic, Claude Opus 4.7 출시 — 1M 컨텍스트 기본 지원
   요약: 1백만 토큰 컨텍스트를 추가 비용 없이 기본 제공. 가격은
   기존 Opus 4.6 대비 동일 유지.
   원문: https://anthropic.com/news/...     (Anthropic Blog · 5/19 06:12 KST)

2. ...

━━ ② 농산물·유통 (3건) ━━━━━━━━━━━━

3. GS리테일, 봄철 신선식품 1차 밴드 운영 정책 일부 조정
   요약: 산지 직배송 비중을 30%→40%로 확대. 1차 밴드 파트너
   납품 기준 일정 단축 발표.
   원문: https://gsretail.com/news/...      (GS리테일 IR · 5/18 17:30 KST)

4. ...

━━ ③ 팜보스 관심 키워드 (2건) ━━━━━━━━

7. 청도 복숭아, 평년 대비 출하 시기 5일 빨라질 전망
   요약: ...
   원문: ...                                (농민신문 · 5/19 05:40 KST)

8. ...

─────────────────────────────────────────
이 메일에 회신해 의견·소스 추가 요청을 보내실 수 있습니다.
운영: 김종만 대표 / 정은주 이사
─────────────────────────────────────────
```

본문 포맷은 Discovery 결론 #1(*카테고리 헤더 + 항목 5~10건 + 항목당 제목·한국어 요약 2 문장 이내·원문 링크*)을 그대로 따른다.

### 2-3. "이상한 날" — 부분 실패·새 뉴스 부재 시

- **소스 12개 중 일부 실패**: 헤더에 "소스 N개 중 M개 정상 수집, X개 실패: {소스명}" 한 줄. 발송은 멈추지 않는다(CLAUDE.md CRITICAL: 외부 소스 장애 격리).
- **카테고리에 새 뉴스 0건**: "오늘 새 뉴스 없음"으로 명시. 발송을 건너뛰지 않는다 — 직원이 "봇이 죽었나"를 의심하지 않게 한다.
- **전체 다이제스트 발송 자체 실패**: 운영자(김종만·정은주)에게 별도 alert 메일이 가야 한다(V1 범위). 직원에게는 발송 안 됨.

### 2-4. 직원 입장에서의 약속

- **시간 약속**: KST 07:30 ± 5분 도착 (월~금 영업일). 4주 연속 95% 정시.
- **신뢰 약속**: 요약 옆에 항상 원문 링크. 의심 시 1클릭으로 검증.
- **중복 없음 약속**: 어제 받은 기사가 오늘 또 오지 않는다.
- **카테고리 약속**: 회사 사업 영역과 연결되지 않는 일반 뉴스는 보내지 않는다. "팜보스 관심 키워드" 카테고리가 그 보장.

## 3. 핵심 규칙

> 구현 방법은 포함하지 않는다. "무엇이 맞는 동작인가"의 규칙만.

### 3-1. 발송 시각·빈도

- **빈도**: 하루 1회.
- **시각**: KST 07:30. **매일 발송** (토·일·공휴일 포함, 2026-05-19 사용자 결정). cron은 KST 07:20에 시작해 정시성 ± 15분 95% 흡수.
- **시간대 표기**: 본문의 모든 시각은 KST 절대 시각(`5월 19일 06:12 KST`). "오늘/어제"는 헤더 1회만 허용(Discovery 결론 #4).

### 3-2. 카테고리

- **고정 3개**: AI 트렌드 / 농산물·유통 / 팜보스 관심 키워드. 순서 고정.
- **카테고리당 5~10건**. 0건이면 "오늘 새 뉴스 없음" 명시 후 카테고리 자체는 표기 유지.
- 한 기사가 두 카테고리에 매칭 가능하면 **더 좁은 카테고리(팜보스 관심 키워드 > 농산물·유통 > AI 트렌드)** 한 곳에만 노출한다.

### 3-3. 항목 형식

- **제목**: 원문 제목 그대로 (한국어 원문은 그대로, 영어 원문은 한국어 번역 제목 + 괄호로 원제 병기).
- **요약**: 한국어 2 문장 이내. 숫자·고유명사·날짜는 원문 그대로 인용. 원문에 명시되지 않은 수치·인과는 생성 금지(Discovery 결론 #2).
- **원문 링크**: URL 전체 노출(메일 클라이언트 자동 link 처리). 짧은 URL 단축 금지(검증성 우선).
- **메타**: 출처명(예: 농민신문), 발행 시각(KST 절대값).
- **"why it matters" 코멘트 없음** (V1, Discovery 결론 #2). V2에서 재검토.

### 3-4. 중복 제거 (dedup)

- 같은 기사가 여러 소스에서 들어오면 **1건만 노출** (가장 권위 있는 출처 우선).
- 같은 URL을 **최근 7일 안에 발송한 적 있으면 노출 금지**.
- URL은 정규화(쿼리스트링·utm 제거) 후 비교. 제목 fuzzy match로 보조 검사.

### 3-5. 실패 격리·운영자 알림

- 한 소스 fetch 실패 → 해당 소스만 제외, 다른 소스 정상 처리. 헤더에 "X개 실패: {이름}" 명시.
- 전체 발송 실패(SMTP·인증·Claude API quota 초과 등) → 운영자 별도 alert(이메일). 직원 메일은 발송하지 않는다.
- Claude API 일일 토큰·호출 hard cap 도달 → 즉시 중단. 운영자 alert.

### 3-6. 수신자

- V1 수신자: 김종만·정은주·장석중 3명 + 일반 직원 N명. 정확한 명단은 `config/recipients.yml`(별도 시크릿 관리).
- 수신자 추가·제거는 코드 변경 없이 yml 파일만 수정.
- 외부 공개 금지(사내 도구).

### 3-7. 비-목표 (명시)

- 슬랙·카카오워크·텔레그램 등 메신저 발송 — V2.
- 사용자별 카테고리 커스터마이즈 — V2 이후.
- 웹 대시보드·과거 다이제스트 검색 UI — V3 이후(Gmail 검색으로 대체 가능).
- 댓글·반응 수집·피드백 루프 — V3 이후.
- 유료 데이터 소스(블룸버그·로이터 등) — 보류.
- 속보 실시간 알림 — 의도적 비-목표(우리는 "출근 직후 1회"만).
- 콘텐츠 자동 작성(마케팅 카피·SNS 글 생성) — 의도적 비-목표.

## 4. 리서치 시사점

Discovery Research(2026-05-19) 결론을 인용해 본 브리프 규칙의 근거를 명시한다.

- 본문 포맷은 검증된 매일 다이제스트 패턴(카테고리 헤더 + 항목 5~10건 + 한국어 요약 2 문장 + 원문 링크) `(discovery-research.md 결론 #1)`
- 요약 신뢰성은 "짧게 + 원문 링크 + 숫자/고유명사 인용 강제" 3축으로 통제. "why it matters" 같은 LLM 코멘트는 V1 도입하지 않음 `(discovery-research.md 결론 #2)`
- 점수화·요약 호출 분리 여부는 Stage 3 Tech Research에서 비용 추정으로 확정 `(discovery-research.md 결론 #3)`
- 모든 시각 표기는 KST 절대 시각으로 통일, cron UTC 환산은 매 라인 주석 `(discovery-research.md 결론 #4)`
- "팜보스 관심 키워드" 카테고리 초기 시드는 [docs/팜보스_회사소개.md](docs/팜보스_회사소개.md) 산지·작목·법인명에서 추출, dry-run 1주일 후 보강 `(discovery-research.md 결론 #5)`

## 5. 미결 항목 — 사용자 일괄 검토 결과 (2026-05-19 해소)

| # | 항목 | 결정 |
|---|---|---|
| 1 | 주말·공휴일 발송 | **매일 발송** (토·일·공휴일 포함). cron `*` 요일 필터. |
| 2 | 수신자 명단·범위 | **단계적 공개** — V1 발송 시작일 운영자 1명만 → 1주 후 김종만·정은주·장석중 3이사 합류 → 2주 후 전 직원. |
| 3 | 회사 내부 선례 | **없음** — 본 V1이 처음. 톤·포맷은 PRD·회사소개 문서 기반으로 에이전트 제안 그대로. |
| 4 | 이모지 사용 | **헤더 이모지 1개(📰) + 카테고리 번호(①②③)**. 본문 항목 내부 이모지는 사용하지 않음. |
| 5 | 외부 뉴스레터 중복 | **권고 안 함, 직원 자율**. 회사 키워드 카테고리는 외부 뉴스레터가 다루지 않으므로 V1 고유 가치. |
| 6 | 운영자 alert 채널 | **별도 메일**. 직원 다이제스트 본문은 깨끗하게 유지, 실패·quota 시 ops_alert 그룹에만 별도 메일. |
