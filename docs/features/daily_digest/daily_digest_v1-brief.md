# daily_digest V1 — 브리프

> 역할: V1 매일 뉴스 다이제스트 봇이 무엇을 하고 무엇을 하지 않는지, 사용자가 무엇을 경험하는지, 어떤 규칙이 그 경험을 만드는지 명시한다. 구현 방법·데이터 스키마는 포함하지 않는다(Stage 4 requirements에서 다룸).
> 대상: Stage 2 Concept 검토, Stage 3 Tech Research 진입자, Stage 4 requirements 작성자, 사용자 일괄 검토자.

---

## 1. 배경

팜보스 그룹(3법인: 정다운 산지유통·팜보스 온라인 커머스·시경 프랜차이즈, 2024년 매출 약 130억) 임직원은 **AI 트렌드**와 **농산물 유통 환경 변화**를 모두 추적해야 한다. 김종만 총괄대표는 3법인 전략을, 정은주 이사는 온라인 사업·마케팅을, 장석중 이사는 산지·GS리테일 납품을 책임진다. 일반 직원은 매일의 업무를 회사 사업 영역과 연결된 정보 안에서 시작해야 한다.

현재는 직원이 개별적으로 여러 사이트(농민신문·식품음료신문·Anthropic Blog·Hacker News 등)를 순회하거나 외부 뉴스레터(TLDR AI·미라클레터 등)에 의지한다. 그 결과:

- **시간 손실**: 매일 30분~1시간이 흩어지는 검색에 쓰인다.
- **편차**: 직원마다 받는 정보가 다르고, 회의 시 같은 기반 위에서 시작하지 못한다.
- **회사 키워드 누락**: 일반 뉴스레터는 "청도 복숭아 시세", "GS리테일 1차 밴드 정책" 같은 회사 직결 키워드를 다루지 않는다.

**V1 다이제스트 봇은 이 세 문제를 동시에 해결한다.** 매일 KST 07:30, 사내 텔레그램 단톡방에 "AI 트렌드 / 농산물 유통 / 팜보스 관심 키워드" 3카테고리의 큐레이션을 짧은 인덱스로 발송하고, 전체 본문 HTML은 GitHub Pages에 매일 1페이지씩 자동 게시한다. 결정은 사람이, 수집·정리·요약은 봇이 한다.

## 2. 사용자 경험

### 2-1. 발송 흐름

```
KST 07:30 (출근 전)
  ↓
[자동] HTML 본문이 GitHub Pages에 게시됨
       → https://{org}.github.io/f_trendnewsbot/digest/2026-05-19.html
[자동] 텔레그램 단톡방에 짧은 인덱스 + Pages 링크 알림 도착
  ↓
직원이 단톡방 push 알림을 본다 (3초)
  "📰 5/19(월) 오늘의 뉴스 8건
   ① AI: Claude Opus 4.7 출시 외 2건
   ② 농산물·유통: GS리테일 산지 직배송 확대 외 2건
   ③ 팜보스: 청도 복숭아 출하 앞당겨 외 1건
   전체: https://...github.io/digest/2026-05-19.html"
  ↓
관심 헤드라인 있으면 Pages 링크 클릭 (브라우저 자동 오픈)
  ↓
HTML 본문에서 "한국어 요약 2 문장 + 원문 링크" 읽기 (5분 분량)
  ↓
업무 시작
```

직원이 단톡방 알림만 보고 그날은 스킵해도 무방. Pages는 매일 URL이 누적되니 며칠 뒤 몰아보기도 가능.

### 2-2. 본문 구조 — 직원이 보는 두 표면

#### 표면 A: 텔레그램 단톡방 메시지 (짧은 인덱스, ~700자)

```
📰 팜보스 트렌드 · 5/19(화) · 8건
✅ 11/12 정상  ⚠️ 한국농어민신문 실패

⚡ 오늘 꼭 챙길 2건
▸ GS리테일, 산지 직배송 30%→40% 확대
   → 정다운 1차 밴드 납품 조건 영향 가능
▸ 청도 복숭아 출하 5일 앞당겨질 전망
   → 정다운 수매·팜보스 사전 예약 일정 점검

① AI 트렌드 (3건)
⭐⭐ Claude Opus 4.7 출시, 1M 컨텍스트 기본
⭐   GPT-5 mini 단가 30% 인하
⭐   Gemini 3 Pro 벤치마크 공개

② 농산물·유통 (3건)
⭐⭐⭐ GS리테일 산지 직배송 30%→40%
⭐⭐  aT 5월 셋째 주 시세 (복숭아 +8.3%, 딸기 -6.1%)
⭐    쿠팡 콜드체인 5개 권역 추가

③ 팜보스 관심 키워드 (2건)
⭐⭐⭐ 청도 복숭아 출하 5일 앞당겨질 전망
⭐⭐  안동농협공판장 정산 주기 5→3영업일 시범

전체 본문·회사 영향 분석:
https://...github.io/digest/2026-05-19.html
의견·소스 제안은 단톡방 답글로.
```

⭐ 표시는 회사 영향 우선순위 (⭐⭐⭐ = 직접 영향 / ⭐⭐ = 간접 / ⭐ = 산업 동향). TL;DR 박스는 ⭐⭐⭐ 자동 추출, 0건이면 "오늘은 산업 동향 위주"로 fallback.

#### 표면 B: GitHub Pages HTML (본문 — 카테고리당 5~10건 풀)

Pages URL: `https://{owner}.github.io/f_trendnewsbot/digest/YYYY-MM-DD.html`

**Pages source = `gh-pages` branch root** (2026-05-19 결정, ADR-003 §대안 F supersede). master branch 의 `docs/canonical/`·`docs/features/`·`docs/_extracted/` 같은 회사 사내 문서는 Pages 에 노출되지 않는다. dispatcher 가 매일 cron 마다 `gh-pages` branch 에 `digest/YYYY-MM-DD.html` 만 commit·push.

디자인 톤은 **애플 사이트 감성 미니멀** (requirements AC-2.7, 2026-05-19 결정). 구성 요소:

1. **Hero** — 큰 타이포(`오늘 챙길 뉴스 N건.` 48~56px Bold, "뉴스 N건"만 blue→purple 그라데이션 강조), 작은 eyebrow(`2026 · 5 · 19 화요일 · KST 07:30`), 한 줄 lede(`AI·농산물 유통 동향, 그리고 팜보스 운영에 직결되는 변화.`), 메타 한 줄(`소스 12개 중 11개 정상 · 한국농어민신문 일시 장애`).
2. **TL;DR 박스** — `⭐⭐⭐` 자동 추출 1~3건. 회색(`#f5f5f7`) 배경 + 18px radius. 1건 이상 있으면 "오늘 꼭 챙길 N건", 0건이면 "오늘은 산업 동향 위주"로 fallback.
3. **카테고리 3개** — 각각 eyebrow `Category 01/02/03` + 36px Bold 제목(`AI 트렌드` / `농산물 · 유통` / `팜보스 관심 키워드`) + "이 카테고리 핵심" 한 줄.
4. **항목** — 우선순위 점 indicator(`••●` / `••○` / `•○○`) + 출처·발행시각 작은 caps + 제목 24px Bold + 본문 17px + `💡 회사 영향:` 한 줄(`#f5f5f7` 배경 박스, 영향 없으면 빈 라인 `회사 직접 영향 없음`) + `원문 읽기 →` Apple Blue 링크.
5. **풋터** — `팜보스 트렌드 · 매일 KST 07:30 · 2026 팜보스 그룹` + hallucination 경고 한 줄.
6. **반응형** — 모바일 600px 이하에서 hero·카테고리 폰트 자동 축소.

샘플 동결: [samples/2026-05-19-digest-preview-v3.html](../../../samples/2026-05-19-digest-preview-v3.html)

HTML head: `<meta name="robots" content="noindex,nofollow">` 명시. 검색엔진 노출 차단.

본문 포맷은 Discovery 결론 #1(*카테고리 헤더 + 항목 5~10건 + 항목당 제목·한국어 요약 2 문장 이내·원문 링크*)을 표면 B(Pages)에서 따른다. 표면 A(텔레그램)는 헤드라인만.

### 2-3. "이상한 날" — 부분 실패·새 뉴스 부재 시

- **소스 12개 중 일부 실패**: 텔레그램 메시지·Pages 헤더 양쪽에 "소스 N개 중 M개 정상 수집, X개 실패: {소스명}" 한 줄. 발송은 멈추지 않는다(CLAUDE.md CRITICAL: 외부 소스 장애 격리).
- **카테고리에 새 뉴스 0건**: "오늘 새 뉴스 없음"으로 명시. 발송을 건너뛰지 않는다 — 직원이 "봇이 죽었나"를 의심하지 않게 한다.
- **전체 다이제스트 발송 자체 실패**: 운영자 전용 텔레그램 chat에 별도 alert이 가야 한다(V1 범위). 직원 단톡방·Pages에는 게시 안 됨.
- **Pages 배포 실패**: 텔레그램 메시지 안 URL이 404가 됨. dispatcher는 Pages 게시 성공을 확인한 뒤에야 단톡방 메시지를 발송. push 실패 시 운영자 alert.

### 2-4. 직원 입장에서의 약속

- **시간 약속**: KST 07:30 ± 15분 도착 (매일, 토·일·공휴일 포함). 4주 연속 95% 정시.
- **신뢰 약속**: Pages 본문에는 요약 옆에 항상 원문 링크. 의심 시 1클릭으로 검증.
- **중복 없음 약속**: 어제 단톡방·Pages에 올라간 기사가 오늘 또 오지 않는다.
- **카테고리 약속**: 회사 사업 영역과 연결되지 않는 일반 뉴스는 보내지 않는다. "팜보스 관심 키워드" 카테고리가 그 보장.
- **보관 약속**: 매일 Pages URL이 누적된다 (`/digest/YYYY-MM-DD.html`). 며칠 못 본 다이제스트도 URL 패턴으로 직접 조회 가능.

## 3. 핵심 규칙

> 구현 방법은 포함하지 않는다. "무엇이 맞는 동작인가"의 규칙만.

### 3-1. 발송 시각·빈도

- **빈도**: 하루 1회.
- **시각**: KST 07:30. **매일 발송** (토·일·공휴일 포함, 2026-05-19 사용자 결정). cron은 KST 07:20에 시작해 정시성 ± 15분 95% 흡수.
- **시간대 표기**: 본문의 모든 시각은 KST 절대 시각(`5월 19일 06:12 KST`). "오늘/어제"는 헤더 1회만 허용(Discovery 결론 #4).

### 3-2. 카테고리

- **고정 3개**: AI 트렌드 / 농산물·유통 / 팜보스 관심 키워드. 순서 고정.
- **카테고리당 5~10건**. 0건이면 "오늘 새 뉴스 없음" 명시 후 카테고리 자체는 표기 유지.
- 한 기사가 두 카테고리에 매칭 가능하면 **더 좁은 카테고리(팜보스 관심 키워드 > 농산물·유통 > AI 트렌드)** 한 곳에만 노출한다.

### 3-3. 항목 형식

**텔레그램 메시지 (표면 A) — 인덱스 + TL;DR**:
- 헤더: `📰 팜보스 트렌드 · M/D(요일) · N건` + 소스 정상/실패 한 줄
- TL;DR 1~3건: 우선순위 ⭐⭐⭐ 항목을 `▸ 제목 → 회사 영향 한 줄`로
- 카테고리당 항목 헤드라인 1줄씩 + 우선순위 `⭐⭐⭐ / ⭐⭐ / ⭐` 표시
- 본문 마지막에 Pages URL 1개만 (개별 원문 링크 노출하지 않음, 채팅창 마찰 최소화)
- 풋터: `의견·소스 제안은 단톡방 답글로`

**GitHub Pages HTML (표면 B) — 전체 본문 + 분석**:
- **제목**: 원문 제목 그대로 (한국어 원문은 그대로, 영어 원문은 한국어 번역 제목 + 괄호로 원제 병기).
- **요약**: 한국어 2 문장 이내. 숫자·고유명사·날짜는 원문 그대로 인용. 원문에 명시되지 않은 수치·인과는 생성 금지(Discovery 결론 #2).
- **회사 영향 한 줄** (2026-05-19 신규): `💡 회사 영향:` prefix + `#f5f5f7` 박스. 회사 사업 영역과 직결되면 어느 법인·어떤 액션 필요한지 1문장. 직결 영향 없으면 빈 라인(`회사 직접 영향 없음`)으로 통일. 억지 추론 금지 (requirements AC-2.5).
- **우선순위 점 indicator** (2026-05-19 신규): `••●` (⭐⭐⭐) / `••○` (⭐⭐) / `•○○` (⭐). 회사 영향 기준 LLM 점수화.
- **원문 링크**: `원문 읽기 →` Apple Blue. URL 전체 보존(짧은 URL 단축 금지).
- **메타**: 출처명·발행 시각(KST 절대값) — 항목 상단 작은 caps eyebrow.
- **HTML head**: `<meta name="robots" content="noindex,nofollow">` 필수. 검색엔진 노출 차단.

### 3-4. 중복 제거 (dedup)

- 같은 기사가 여러 소스에서 들어오면 **1건만 노출** (가장 권위 있는 출처 우선).
- 같은 URL을 **최근 7일 안에 발송한 적 있으면 노출 금지**.
- URL은 정규화(쿼리스트링·utm 제거) 후 비교. 제목 fuzzy match로 보조 검사.

### 3-5. 실패 격리·운영자 알림

- 한 소스 fetch 실패 → 해당 소스만 제외, 다른 소스 정상 처리. 단톡방 메시지·Pages 헤더에 "X개 실패: {이름}" 명시.
- Pages 게시 실패(git push 실수·인증 오류) → 단톡방 메시지 발송 안 함. 운영자 alert.
- 전체 발송 실패(텔레그램 Bot API 인증·Claude API quota 초과 등) → 운영자 전용 텔레그램 chat에 별도 alert. 직원 단톡방은 게시 안 됨.
- Claude API 일일 토큰·호출 hard cap 도달 → 즉시 중단. 운영자 alert.

### 3-6. 수신자

- V1 직원 수신: 사내 텔레그램 단톡방 1개 (다이제스트 전용). 단톡방 멤버가 수신자.
- 수신자 추가·제거 = 단톡방 입퇴장으로 처리. `config/recipients.yml` 같은 별도 파일 불필요.
- 운영자 alert 수신: 운영자 본인 텔레그램 chat 또는 운영자 전용 단톡방 1개.
- GitHub Pages는 public + noindex이므로 URL 알면 누구나 접근 가능. 사내 정보 본문 금지(시세·매출·인사 등). 본문은 외부 뉴스 큐레이션만.
- 외부 노출 회피: HTML head noindex meta + `docs/digest/robots.txt`로 크롤러 차단.

### 3-7. 비-목표 (명시)

- 이메일 발송 — V1에서 제거 (ADR-003, 2026-05-19 결정). V2 이후 이사진 요청 시 재도입 검토.
- 슬랙·카카오워크·Teams 메신저 발송 — V2 이후 (텔레그램이 1차 채널).
- 카카오톡 일반 메시지·알림톡 발송 — 봇 API 폐쇄적이라 V2에서도 미적용.
- 회사 영향이 아닌 일반 광고형 코멘트("Why it matters", "당신에게 의미" 등) — V1에서 명시적으로 금지. `💡 회사 영향:` 라인만 허용하되 회사 사업 영역 직결 영향이 없으면 빈 메시지(`회사 직접 영향 없음`)로 통일 — 억지 추론 금지 (requirements AC-2.5).
- 사용자별 카테고리 커스터마이즈 — V2 이후.
- 검색 UI·과거 다이제스트 검색 — V3 이후. V1은 Pages URL 패턴(`/digest/YYYY-MM-DD.html`)로 직접 조회.
- 댓글·반응 수집·피드백 루프 — V3 이후.
- 유료 데이터 소스(블룸버그·로이터 등) — 보류.
- 속보 실시간 알림 — 의도적 비-목표(우리는 "출근 직후 1회"만).
- 콘텐츠 자동 작성(마케팅 카피·SNS 글 생성) — 의도적 비-목표.

## 4. 리서치 시사점

Discovery Research(2026-05-19) 결론을 인용해 본 브리프 규칙의 근거를 명시한다.

- 본문 포맷은 검증된 매일 다이제스트 패턴(카테고리 헤더 + 항목 5~10건 + 한국어 요약 2 문장 + 원문 링크) `(discovery-research.md 결론 #1)`
- 요약 신뢰성은 "짧게 + 원문 링크 + 숫자/고유명사 인용 강제" 3축으로 통제. "why it matters" 같은 LLM 코멘트는 V1 도입하지 않음 `(discovery-research.md 결론 #2)`
- 점수화·요약 호출 분리 여부는 Stage 3 Tech Research에서 비용 추정으로 확정 `(discovery-research.md 결론 #3)`
- 모든 시각 표기는 KST 절대 시각으로 통일, cron UTC 환산은 매 라인 주석 `(discovery-research.md 결론 #4)`
- "팜보스 관심 키워드" 카테고리 초기 시드는 [docs/팜보스_회사소개.md](docs/팜보스_회사소개.md) 산지·작목·법인명에서 추출, dry-run 1주일 후 보강 `(discovery-research.md 결론 #5)`
