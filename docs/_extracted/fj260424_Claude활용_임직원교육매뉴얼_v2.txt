1부. 왜 Claude를 배우는가
우리 회사의 현실
팜보스 그룹은 3개 법인(팜보스·정다운·시경)을 통합 운영하며, 2024년 매출 130억원 규모로 성장했습니다. 그러나 인원은 이 규모의 다른 회사 대비 적습니다. 이는 강점이자 약점입니다.
적은 인원으로 큰 회사를 운영하려면, 반복·단순 업무는 AI가 처리하고 사람은 판단·관계·창의에 집중해야 합니다. 이게 우리가 Claude를 배워야 하는 본질적 이유입니다.
Claude로 달라지는 것
문서 작성: 이메일·공문·보고서 초안을 10분 → 2분
데이터 정리: 흩어진 파일·영수증·메모를 정리된 스프레드시트로
분석·판단: 복잡한 수치·계약서를 빠르게 요약, 리스크 포인트 찾기
반복 업무: 정기 보고·발주서·공지 자동 생성
산지 소통: 농가 응대 문자·공지사항 초안
경영 의사결정: 복수 옵션 비교, 장단점 분석
주의 — Claude는 만능이 아니다
⚠ 반드시 기억하세요
1) Claude는 실수할 수 있습니다. 중요한 수치·날짜·사람 이름은 반드시 사람이 최종 확인해야 합니다.
2) Claude는 법률·의료·세무 상담을 대체하지 못합니다. 참고용으로만 사용하고 전문가 자문을 받으세요.
3) 개인정보·고객정보는 함부로 입력하지 마세요. 주민등록번호·계좌번호·비밀번호 등은 절대 금물.
4) 외부에 나갈 공식 문서는 반드시 사람이 검토 후 내보냅니다.


2부. Claude 두 얼굴 — 데스크톱 앱과 웹의 역할
핵심 개념
Claude는 같은 회사(Anthropic)에서 만든 같은 인공지능이지만, 우리가 접근하는 방법에 따라 두 가지 얼굴을 가집니다. 이 둘을 구분해서 사용하는 게 이 매뉴얼의 핵심입니다.
얼굴 1: 웹 채팅 (브라우저로 접속)
주소: claude.ai
사용 방법: 크롬·엣지 등 브라우저에서 접속
특징: 대화·질문·문서 작성·분석에 특화
장점: 가볍고 빠름, 어디서나 접속, 바탕화면 바로가기로 앱처럼 사용
약점: 내 PC의 파일을 직접 건드리지 못함
얼굴 2: 데스크톱 앱의 Cowork 기능
설치: 내 PC에 Claude Desktop 앱 설치
사용 방법: 앱을 열고 좌측에서 'Cowork' 탭 선택
특징: 내 PC의 폴더를 직접 읽고, 수정하고, 새 파일 생성
장점: 파일 정리·문서 생성·이미지→엑셀 자동화 등 실제 작업 완성
약점: 무겁고 초기 설정 필요 (Windows는 가상화 기능 2개 활성화)
왜 두 개를 같이 쓰는가
하나로 다 하려 하면 답답합니다. 두 개를 역할 분담해서 쓰면 각자의 강점이 살아납니다.

💡 우리 회사의 표준 방식
바탕화면에 두 개의 아이콘을 둡니다:
  1) Claude 채팅 (웹 바로가기) — 일상 대화·질문·초안 작성
  2) Claude 데스크톱 (Cowork 탑재) — 파일 작업·자동화

상황에 따라 둘을 동시에 열어놓고 쓰는 게 표준입니다.
예: 데스크톱 앱의 Cowork가 파일 정리하는 동안, 웹 채팅으로는 고객 응대 문자를 작성.


3부. 환경 준비 — 내 PC에서 어떻게 시작하나
여러분의 PC가 어떤 환경이냐에 따라 사용 가능한 기능이 달라집니다. 자기 PC가 어느 경우에 해당하는지 먼저 확인하세요.
3.1 내 PC 확인하는 법
Windows 사용자
키보드에서 Windows 키 + R을 누릅니다.
열린 창에 'winver'를 입력하고 엔터.
뜨는 창에서 Windows 버전을 확인합니다.
또는 설정 → 시스템 → 정보에서 '에디션' 확인.
Mac 사용자
좌측 상단 Apple 로고 → 'About This Mac' 클릭.
macOS 버전 확인 (10.15 Catalina 이상이어야 함).
3.2 경우별 시나리오
경우 1: Windows 11 (Home·Pro·Enterprise 모두 동일)
🎯 에디션에 관계없이 Cowork 사용 가능합니다.
✅ 웹 채팅 사용 가능
✅ 데스크톱 앱 Cowork 사용 가능

※ 단, 'Virtual Machine Platform'과 'Windows Hypervisor Platform'
   두 기능을 Windows에서 직접 켜야 합니다 (3.3 참조).
   기본값은 꺼져 있습니다 — Pro여도 켜야 작동합니다.
 
경우 2: Mac (macOS 10.15 이상)
🎯 별도 설정 없이 모든 기능 사용 가능.
✅ 웹 채팅 사용 가능
✅ 데스크톱 앱 Cowork 사용 가능
→ 3.4의 Mac 설치 절차 참고.
 
경우 3: 회사 지급 노트북 (관리자 권한 제한)
⚠ 일부 회사 노트북은 앱 설치·기능 활성화 권한이 제한되어 있습니다.
✅ 웹 채팅은 브라우저만 있으면 사용 가능 (문제 없음)
△ Cowork 설정은 관리자 권한 필요 — 경영지원팀 또는 김대표에게 요청

→ 우선 웹 채팅부터 시작하세요.
 
3.3 Cowork 활성화 절차 (Windows 공통 — Home·Pro 동일)
🔑 핵심 사실
Cowork 작동에는 Windows 에디션(Home·Pro)이 아니라,
아래 두 기능의 활성화 여부가 결정적입니다.

  ① Virtual Machine Platform (가상 머신 플랫폼)
  ② Windows Hypervisor Platform (Windows 하이퍼바이저 플랫폼)

이 두 기능은 Home·Pro·Enterprise·Education 모든 에디션에서 제공됩니다.
기본값은 꺼져 있으니 반드시 직접 켜야 합니다.

⚠ Hyper-V(별도 항목)와 혼동 주의 — Hyper-V는 Pro 이상 전용이지만,
    Cowork 작동에는 Hyper-V가 필요 없습니다.
 
1단계: Windows 기능 창 열기
키보드에서 Windows 키 + R 누르기.
실행 창에 'optionalfeatures' 입력 → 엔터.
(또는 제어판 → 프로그램 → '프로그램 및 기능' → 좌측 'Windows 기능 켜기/끄기')
'Windows 기능' 목록 창이 열림.
2단계: 두 항목 체크
목록을 알파벳 순으로 스크롤.
'Virtual Machine Platform' (가상 머신 플랫폼) 앞 체크박스 ☑ 클릭.
'Windows Hypervisor Platform' (Windows 하이퍼바이저 플랫폼) 앞 체크박스 ☑ 클릭.
하단 '확인' 클릭.
'필요한 파일 검색 중' → '변경 내용 적용 중' 진행바 대기 (2~5분).
'지금 다시 시작' 버튼 클릭 → PC 재부팅.
3단계: Claude Desktop 앱 실행 확인
재부팅 완료 후 Claude 데스크톱 앱 실행.
좌측 사이드바에 'Cowork' 탭이 보이는지 확인.
첫 실행 시 약 2GB 가상머신 이미지 자동 다운로드 (5~15분).
다운로드 완료 후 Cowork 작업 창이 열리면 성공.
작동이 안 될 때: BIOS 가상화 확인
Windows 기능 두 개를 켰는데도 Cowork가 작동하지 않는 경우는 대부분 BIOS(UEFI) 단계에서 하드웨어 가상화가 꺼져 있기 때문입니다.
[먼저 현재 상태 확인]
Ctrl + Shift + Esc → 작업 관리자.
'성능' 탭 → CPU 선택.
하단 '가상화' 항목이 '사용'이어야 함.
'사용 안 함'이면 아래 BIOS 설정 변경 필요.
[BIOS 가상화 활성화]
설정 → 시스템 → 복구 → '고급 시작 옵션' → '지금 다시 시작'.
파란 화면 → '문제 해결' → '고급 옵션' → 'UEFI 펌웨어 설정' → '다시 시작'.
BIOS 진입 후 'Configuration' 또는 'Advanced' 탭.
'Intel Virtualization Technology (VT-x)' 또는 'AMD-V' / 'SVM Mode' → Enabled.
F10 또는 'Save & Exit'로 저장 후 재부팅.
※ 제조사별(Lenovo·HP·Dell·Samsung·LG) 메뉴 명칭이 다릅니다. 공통 키워드: Virtualization, VT, SVM.
그래도 안 되는 경우
⚠ 드문 케이스지만 발생 가능
• 매우 오래된 CPU (2012년 이전 출시) — 하드웨어 가상화 미지원
• 회사 보안 정책으로 가상화가 원천 차단된 일부 업무용 노트북
• Windows 기능 설치 중 오류 (일반적으로 Windows 업데이트 후 재시도하면 해결)

이런 경우엔 해당 PC에서 Cowork 포기하고, 웹 채팅만 사용하거나
회사 공용 Cowork PC를 활용하세요 (3.5 참조).
 
3.4 Mac에서 Cowork 활성화
Mac은 별도 Windows 기능 설정이 필요 없습니다. macOS 자체에 가상화가 내장되어 있어, Claude Desktop 앱을 설치하면 바로 Cowork 탭이 표시됩니다.
claude.ai/download 에서 macOS용 데스크톱 앱 다운로드.
.dmg 파일 실행 → Applications 폴더로 드래그.
Launchpad에서 Claude 실행.
회사 계정으로 로그인.
좌측 사이드바에서 'Cowork' 탭 확인.
첫 실행 시 약 2GB 가상머신 이미지 자동 다운로드.
3.5 Cowork를 쓸 수 없는 PC에서의 대안
옵션 A: 웹 채팅만 사용
Cowork 없이도 Claude의 80% 기능은 웹 채팅으로 해결됩니다. 문서 작성·이메일·분석·번역·요약·초안 작성 등 일상 업무 대부분은 웹 채팅으로 충분합니다.
Cowork가 꼭 필요한 업무는 파일 자동 정리·대량 이미지 처리·폴더 통째 변환 같은 특수한 경우입니다. 본인 업무가 여기에 해당하지 않으면 웹 채팅만으로 충분합니다.
옵션 B: 공용 Cowork PC 활용
회사 내 Cowork 활성화된 PC를 한 대 공용으로 두고, 파일 일괄 처리·대량 작업 시 해당 PC에서 작업. 평소엔 각자 PC에서 웹 채팅 사용.
관리: 경영지원팀이 운영, 예약제 권장
설치: 해당 PC에 3.3 절차 적용 후 전사 공용 아이디로 로그인
💡 김대표 권장 순서
1단계 (지금 즉시): 본인 PC에서 3.3 절차로 Virtual Machine Platform·
                    Windows Hypervisor Platform 두 기능 활성화 시도

2단계: 재부팅 후 Claude 앱에 Cowork 탭 표시되면 → 완료
        안 보이면 → BIOS 가상화 확인 (3.3의 마지막 단계)

3단계: 그래도 안 되면 → 웹 채팅만으로 먼저 1개월 활용 후 필요성 재평가

※ Windows Home이든 Pro든 동일한 절차입니다. 에디션 업그레이드 불필요.


4부. 바탕화면 바로가기 만들기 (웹 채팅용)
웹 채팅(claude.ai)을 매번 브라우저 열고 주소 치지 말고, 바탕화면에 아이콘을 만들어 앱처럼 사용하세요. 이게 Claude를 가장 빠르게 여는 방법입니다.
4.1 Chrome 브라우저 사용자 (권장)
방법 1: PWA(앱 설치) 방식 — 가장 추천
Chrome을 열고 claude.ai에 접속해 로그인합니다.
주소창 오른쪽 끝의 '▼ 설치' 아이콘을 찾습니다. (Chrome 버전에 따라 ⋮ 메뉴 → '앱으로 설치' 또는 '캐스트, 저장 및 공유' 안에 있음)
'설치' 클릭 → 팝업에서 '설치' 재확인.
바탕화면과 시작 메뉴에 Claude 아이콘이 자동 생성됩니다.
이후엔 아이콘 더블클릭하면 독립된 창에서 Claude가 열립니다 (브라우저 탭이 아닌 앱처럼).
✅ PWA 방식의 장점
• 전용 창으로 열림 (브라우저 주소창 없음)
• 작업 표시줄에 별도 앱처럼 고정 가능
• Alt+Tab으로 빠르게 전환
• 알림 수신 가능 (브라우저 설정에 따라)
• 바로 Claude에 접근 — 3초면 시작
 
방법 2: 단순 바로가기
바탕화면 빈 공간 우클릭 → '새로 만들기' → '바로 가기'.
위치 입력란에 다음 붙여넣기: https://claude.ai
'다음' → 이름에 '클로드 채팅' 입력 → '마침'.
바탕화면에 아이콘 생성 완료.
4.2 Edge 브라우저 사용자
Edge를 열고 claude.ai 접속·로그인.
주소창 오른쪽 ⋯ 메뉴 → '앱' → '이 사이트를 앱으로 설치'.
이름 '클로드 채팅' 입력 → '설치'.
옵션에서 '바탕 화면에 바로가기 만들기' 체크 → '허용'.
4.3 아이콘 정리 팁
이름 표준화: '클로드 채팅' 또는 'Claude-Web'으로 통일 (사내 공용)
위치: 바탕화면 좌측 상단 또는 작업 표시줄에 고정

5부. 지침 설정 (User Preferences)
Claude는 여러분이 누구인지, 어떤 스타일로 답변받고 싶은지를 미리 알려주면 매번 설명하지 않아도 맞춤 답변을 줍니다. 이걸 '지침(Preferences)'이라고 합니다.
5.1 지침이란 무엇인가
지침은 Claude에게 미리 알려두는 '내 배경 정보와 대화 규칙'입니다. 한 번 설정하면 이후 모든 대화에 자동 적용됩니다.
좋은 비유: 새 직원이 출근하기 전에 회사 소개서를 미리 읽는 것과 같습니다. 회사가 뭘 하는지, 제가 누군지, 어떻게 답변받고 싶은지 미리 알려주면 매번 반복 설명할 필요가 없습니다.
5.2 지침 설정 위치
웹 채팅(claude.ai)의 경우
좌측 하단 본인 이름·프로필 아이콘 클릭.
'Settings' 또는 '설정'.
'Profile' 또는 '프로필' 탭.
'What would you like Claude to know about you?' 또는 '개인 환경설정' 입력란.
내용 입력 후 저장.
데스크톱 앱의 경우
앱 좌측 하단 설정 아이콘.
설정 → 프로필(또는 Preferences) 섹션.
내용 입력 후 저장. 웹과 별도로 관리될 수 있으니 양쪽 모두 동기화 확인.

6부. 메모리 활용법
6.1 메모리란
메모리는 Claude가 여러분과 나눈 과거 대화에서 배운 정보를 기억해두는 시스템입니다. 같은 이야기를 매번 반복할 필요가 없어집니다.
예를 들어 '저는 팜보스 그룹 산지 MD 김철수입니다. 주 담당은 복숭아입니다'라고 한 번 말하면, 다음 대화에서 '우리 담당 품목 단가 정리해줘'라고만 해도 Claude가 '복숭아 말씀이시죠?'라고 반응합니다.

6.3 메모리 설정·관리
메모리 켜기
설정 → '기능' 또는 'Features' 탭.
'채팅 기록에서 기억 생성' 토글 켜기.
'채팅 검색 및 참조'도 함께 켜면 과거 대화까지 검색 가능.
메모리 보기·수정
같은 설정 페이지에서 '채팅에서 얻은 메모리' 영역 확인.
미리보기로 Claude가 기억하고 있는 내용 확인 가능.
잘못 저장된 내용은 직접 수정·삭제 가능.
메모리 요청 명령어
대화 중 직접 메모리에 저장하거나 잊도록 요청할 수 있습니다:
'이걸 기억해줘: 복숭아 산지는 경북 청도 반포리.'
→ Claude가 해당 정보를 메모리에 등록합니다.
 
'이전에 말한 A거래처 정보는 잊어줘.'
→ Claude가 해당 정보를 메모리에서 제거합니다.
 
'지금까지 나에 대해 기억하는 걸 다 보여줘.'
→ Claude가 현재 메모리 내용을 요약해서 보여줍니다.
6.4 팜보스 메모리 활용 우수 사례
사례 1: 반복 질문 제거
매주 정기 보고를 Claude와 함께 작성하는 직원의 경우, 첫 주에 보고 양식과 규칙을 자세히 설명하면 다음 주부터는 '이번주 보고 작성' 한 마디로 동일 양식이 적용됩니다.
사례 2: 거래처 맥락 유지
산지 MD가 주요 거래처 20곳의 특성(선호 품종·결제 조건·연락 스타일)을 메모리에 저장해두면, 특정 거래처 응대 문자를 만들 때 즉시 맞춤 톤으로 생성됩니다.
사례 3: 프로젝트 연속성
긴 프로젝트(예: 신규 매장 오픈)의 진행 상황을 메모리에 누적하면, 2주 후 다시 대화해도 맥락이 끊기지 않습니다.
6.5 메모리 보안 주의사항
⚠ 메모리에 저장하면 안 되는 것
• 주민등록번호·계좌번호·비밀번호
• 고객 개인정보 (이름+연락처+구매내역 조합)
• 미공개 재무 수치, 영업비밀의 핵심
• 인사 평가 상세 내용 (평가 대상자 이름 포함)

이런 정보는 그 대화에서만 사용하고 끝내야 합니다.
메모리에 들어가면 이후 다른 대화에서 실수로 노출될 수 있습니다.


7부. 스킬(Skills) 사용법

스킬을 켜두면 Claude가 알아서 필요한 기술을 골라 씁니다. 여러분이 매번 지정할 필요 없습니다.

예시 3: 교육자료 PPT 만들기
요청: '신입직원 대상 팜보스 그룹 소개 PPT 10장짜리 만들어줘. 법인별 역할·매출 추이·조직도 포함.'
결과: pptx 스킬 실행, 편집 가능한 .pptx 파일 생성.
7.5 스킬은 Cowork에서 더 강력
스킬은 웹 채팅에서도 쓰지만, Cowork에서 쓰면 한 단계 더 강력해집니다. 폴더를 통째로 넘겨서 'PDF 10개를 하나의 엑셀로 정리'같은 작업이 가능해집니다.

8.2 작업 폴더 지정
Cowork는 지정한 폴더 안의 파일만 읽고 수정합니다. 이건 보안 장치입니다 - 실수로 중요 시스템 파일을 건드리지 못하게 합니다.

├── inbox\           ← 처리할 파일 넣는 곳
├── processing\      ← Claude가 작업 중
├── output\          ← 완성된 결과물
└── archive\         ← 보관용
 
Mac 사용자:

주의: 지정하면 안 되는 폴더
시스템 폴더: C:\Windows, C:\Program Files — 절대 금지

민감 폴더: 급여·인사·재무 원본 — 백업 없이 노출 금지
자동화 인프라: 사내 봇 스크립트 폴더 — 대표 지정 폴더만 사용
8.2.1 Google Drive와 Cowork 함께 쓰기 (중요)
팜보스 그룹은 많은 업무 파일을 Google Drive에서 관리합니다. Cowork는 PC 안의 로컬 파일만 읽을 수 있기 때문에, Drive의 파일을 Cowork에서 쓰려면 '미러링(mirror)'이라는 방법으로 PC에 연결해둬야 합니다.

🎯 결론
Cowork에서 Google Drive 파일을 쓰려면 반드시 '미러링' 모드여야 합니다.
스트리밍 모드의 Drive 폴더를 Cowork 작업 폴더로 지정하면, Claude가 파일을 제대로 못 읽거나 동기화 충돌이 생깁니다.

다만 미러링은 PC 용량을 실제로 차지하므로, 필요한 폴더만 선택해서 미러링해야 합니다.
 
용량 문제 — 왜 전체 미러링은 안 되나
팜보스 그룹의 공용 Google Drive는 전체 용량이 수백 GB~TB 규모입니다. 이걸 전부 미러링하면 일반 노트북 SSD(256GB·512GB)가 순식간에 가득 차서 PC가 느려지거나 멈춥니다.
해결책은 '선택적 미러링' — 본인이 자주 쓰는 폴더만 골라서 PC에 내려받는 방식입니다.
선택적 미러링 설정 방법 (Windows·Mac 공통)
[사전 준비]
Google Drive for Desktop 설치 완료 (www.google.com/drive/download).
팜보스 회사 계정으로 로그인.
[단계별 설정]
작업 표시줄(Mac: 상단 메뉴바) 우측 Google Drive 아이콘 클릭.
톱니바퀴 아이콘 → '기본 설정' 또는 'Preferences'.
좌측에서 'Google Drive' 또는 '내 드라이브' 탭 선택.
동기화 옵션에서 '파일 미러링' 선택 (스트리밍 아님).
'폴더 옵션' 또는 '특정 폴더 동기화' 선택.
미러링할 폴더만 체크박스로 선택. 나머지는 클라우드에만 둠.
'저장' → Drive가 선택한 폴더만 PC로 내려받기 시작.

💡 용량 관리 팁
• 본인 PC SSD의 50% 이상을 Drive 미러링에 쓰지 마세요. (PC 자체가 느려짐)
• 256GB SSD: 미러링은 100GB 이하
• 512GB SSD: 미러링은 200GB 이하
• 1TB SSD: 미러링은 400GB 이하

• 임시로 필요한 폴더는 쓰고 나서 미러링 해제 (체크박스 해제)
• 해제해도 Google Drive 원본은 그대로 남음 — 필요할 때 다시 미러링
• 오래된 프로젝트 폴더는 일정 주기로 미러링 정리
 
미러링된 폴더를 Cowork에 연결
미러링 완료 후엔 미러링된 Drive 폴더를 Cowork 작업 폴더로 지정하면 됩니다.
[Windows 미러링 Drive 경로 예시]
G:\내 드라이브\팜보스공용\산지거래처\
또는
C:\Users\본인이름\My Drive\팜보스공용\산지거래처\
 
[Mac 미러링 Drive 경로 예시]
~/Library/CloudStorage/GoogleDrive-본인이메일/My Drive/팜보스공용/산지거래처/
 
※ 설치 시 설정한 위치에 따라 다름.
설정 → 'Google Drive 폴더 위치'에서 확인 가능.
 
이 경로를 Cowork의 '작업 폴더'로 지정하면, Cowork가 Drive 파일을 로컬처럼 읽고 쓰고, 바뀐 내용은 자동으로 클라우드에 동기화됩니다.
미러링 + Cowork 동시 사용 시 주의사항
⚠ 반드시 지킬 것
1) Cowork 작업 중에는 다른 PC·폰에서 같은 파일 수정하지 않기 (충돌 방지)
2) 인터넷 끊긴 상태에서 수정한 파일은 연결되면 자동 동기화 — 기다릴 것
3) 공용 폴더(팜보스공용/) 수정은 사전에 팀에 공유 — 남의 파일 덮어쓰기 위험
4) Cowork에게 '파일 삭제'를 지시할 때 특히 주의 — Drive 원본까지 같이 삭제됨
5) 중요 원본은 별도 '보관' 폴더로 복사해두고 작업 (복구 불가 대비)
 
문제 해결 FAQ
Q. 미러링으로 바꿨더니 Drive 파일이 다 사라졌어요
A. 정상입니다. 모드를 바꾸면 Drive가 초기화되면서 기본값(일부만 동기화)이 적용됩니다. 위의 '선택적 미러링 설정'에서 필요한 폴더를 다시 체크해 주세요. 원본은 클라우드에 그대로 있습니다.
Q. SSD가 가득 찼다고 경고가 떠요
A. 미러링 폴더가 너무 많습니다. Drive 설정 → 폴더 옵션에서 불필요한 폴더의 체크를 해제하세요. 해제하면 PC 용량은 비우고, 클라우드 원본은 유지됩니다.
Q. Cowork가 Drive 폴더를 '읽기 전용'으로 인식해요
A. 스트리밍 모드일 때 발생하는 현상입니다. 설정에서 미러링 모드로 전환하거나, Cowork 작업 폴더를 미러링된 다른 경로로 지정하세요.
Q. 미러링하니 PC가 느려졌어요
A. 초기 동기화 중이라 그렇습니다. 첫 1~2시간 정도는 대량 다운로드·색인 생성으로 느립니다. 취침 전에 설정해두고 아침에 확인하는 걸 권장. 이후로는 변경분만 동기화하므로 부담 없습니다.
8.3 첫 작업 시도 - 5분 튜토리얼
단계 1: 테스트 폴더 만들기
바탕화면에 'Claude-test'라는 폴더를 만듭니다. 이 안에 섞어놓은 파일 5~10개 넣기 (오래된 메모, 영수증 사진, 다운로드한 자료 아무거나).
단계 2: Cowork에서 폴더 지정
Cowork 탭 → '작업 폴더 설정' 버튼.
바탕화면의 'Claude-test' 폴더 선택.
권한 요청 창 뜨면 '허용'.
단계 3: 간단한 작업 지시
작업 창에 다음과 같이 입력:
이 폴더 안의 파일들을 날짜별로 분류하고,
파일명이 이상한 것들(예: 'aaa.pdf', 'Untitled1.png')은
내용을 봐서 의미있는 이름으로 바꿔줘.
결과는 'sorted' 폴더에 정리해서 넣어줘.
단계 4: Claude의 계획 확인
Claude는 바로 실행하지 않고 먼저 '계획'을 보여줍니다. 어떤 파일을 읽고, 어떻게 정리하고, 어디에 저장할지 순서대로 설명합니다. 이걸 검토하고 '시작' 또는 '수정 요청'.
단계 5: 진행 관찰
Claude가 실제 작업하는 과정을 볼 수 있습니다. 파일 읽기 → 내용 분석 → 이름 변경 → 이동. 중간에 멈추거나 지시 추가 가능.
단계 6: 결과 확인
완료되면 'sorted' 폴더 열어서 확인. 마음에 안 들면 '되돌려줘' 한 마디로 원복 요청 가능.
8.4 Cowork 핵심 원칙 - 반드시 지키세요
🛡 안전 5대 원칙
1) 중요 폴더는 반드시 백업 후 Cowork 지정 (Google Drive, 외장 HDD)
2) 처음엔 테스트 폴더로 익숙해진 뒤 실제 업무 폴더로 확장
3) Claude의 '계획'을 꼼꼼히 읽고 승인 — 그냥 '시작' 누르지 말기
4) 삭제 요청은 특별히 주의. Claude는 삭제 전 재확인하지만 원복 불가
5) 작업 끝나면 결과물 직접 확인, 이상 있으면 즉시 '되돌려줘' 지시
 
8.5 Cowork 업무 사례 (팜보스 그룹)
사례 A: 영수증 사진 → 엑셀
현장에서 찍은 매입 영수증 사진 30장이 폴더에 있을 때. Cowork에 '이 사진들을 읽어서 날짜·거래처·금액·품목을 엑셀로 정리해줘'라고 지시 → 자동 완성. 30분 수동 작업이 5분.
사례 B: 회의록 여러 개 → 보고서
주간 회의록 4주치(.docx 4개)가 있을 때. '이 4주치 회의록에서 공통 안건과 변화 추이를 뽑아서 월간 보고서로 정리해줘' → 구조화된 보고서 자동 생성.
사례 C: 파일명 표준화
다운로드 폴더에 영업자료 PDF 50개가 제멋대로 이름으로 있을 때. '이 PDF들의 내용을 보고 [업체명][날짜][제목] 형식으로 이름을 통일해줘' → 일괄 정리.
사례 D: 급여명세서 일괄 생성
월급 지급용 명세서 20개를 각 직원별로 따로 만들어야 할 때. 급여 데이터 엑셀 1개 + 양식 템플릿 1개만 주면 Cowork가 직원별 PDF 20개 자동 생성.


9부. 두 창 동시 사용 실전 워크플로우
이 챕터가 매뉴얼의 핵심입니다. 웹 채팅과 Cowork를 동시에 열어놓고 쓰는 실전 방법을 배웁니다.
9.1 왜 동시에?
두 창을 동시에 열어두면 서로의 약점을 보완합니다. Cowork가 무거운 작업(파일 일괄 처리) 하는 동안, 웹 채팅으로 다른 업무(이메일 답장) 진행 가능. 시간 활용 2배.
9.2 기본 배치 - 듀얼 모니터가 없어도 OK
화면 분할 방법 (Windows)
데스크톱 앱(Cowork)을 열고 Windows 키 + ← 누름 → 화면 왼쪽 반 차지.
웹 채팅을 열고 Windows 키 + → 누름 → 화면 오른쪽 반 차지.
두 창이 나란히 배치됨.
화면 분할 방법 (Mac)
데스크톱 앱 창 좌측 상단 녹색 버튼 길게 누름 → '화면 왼쪽 타일'.
빈 오른쪽에 웹 채팅 브라우저 선택.
Split View로 동시 표시.
9.3 워크플로우 패턴 5가지
패턴 1: 병렬 처리 (Cowork 작업 중 + 웹 채팅 다른 일)
상황: 영수증 30장을 엑셀로 정리하는 동안 다른 업무 하기
[왼쪽 Cowork] '영수증 폴더 → 엑셀 정리해줘' 지시 → 실행 중 (5~10분)
[오른쪽 웹 채팅] 그 사이 거래처 응대 문자 2개 작성
→ 10분에 2가지 업무 동시 완료
 
패턴 2: 설계·실행 분리 (웹에서 설계 → Cowork에서 실행)
상황: 복잡한 작업을 먼저 구상한 뒤 실행
[왼쪽 웹 채팅] '신규 매장 오픈 체크리스트와 공지 메일 양식 만들어줘' → 대화로 구조 완성
[오른쪽 Cowork] 웹에서 완성한 양식을 실제 파일(docx·pptx)로 생성하고 관련 폴더에 배치
→ 머릿속 아이디어 → 완성 파일까지 30분
 
패턴 3: 분석·작성 협업 (Cowork에서 분석 → 웹에서 글쓰기)
상황: 데이터를 분석한 뒤 보고서 작성
[왼쪽 Cowork] '지난 6개월 매입 엑셀들 → 거래처별 단가 추이 분석' → 분석 결과 파일 생성
[오른쪽 웹 채팅] 분석 결과를 복사해서 '이 데이터로 경영진 보고용 3줄 요약 만들어줘'
→ 숫자에서 의사결정용 문장까지 원스톱
 
패턴 4: 교차 검증 (웹과 Cowork가 서로 확인)
상황: 중요한 작업의 실수 방지
[오른쪽 웹 채팅] '이 계약서 초안을 검토해줘. 리스크 포인트와 누락사항 알려줘'
[왼쪽 Cowork] 같은 계약서를 'PDF로 변환하고 과거 계약서 폴더와 비교해줘'
→ 두 각도의 검증으로 빠뜨리는 부분 최소화
 
패턴 5: 학습·적용 (웹에서 배우기 → Cowork에서 해보기)
상황: 새 업무 방식을 익힐 때
[오른쪽 웹 채팅] '엑셀 피벗 테이블 만드는 법 알려줘'
[왼쪽 Cowork] 배운 대로 실제 우리 회사 데이터로 피벗 테이블 생성
→ 이론 학습과 실습을 동시에, 한 화면에서

11부. 자주 묻는 질문 (FAQ)
설치·환경
Q1. 내 PC가 Windows Home인데 Cowork가 안 보여요.
A. Windows 에디션(Home·Pro) 문제가 아닙니다. 'Virtual Machine Platform'과 'Windows Hypervisor Platform' 두 기능이 꺼져 있어서입니다. 제어판 → 프로그램 → 'Windows 기능 켜기/끄기'에서 두 항목을 체크하고 재부팅하면 Home에서도 Cowork가 작동합니다. 자세한 절차는 3.3 참조.
Q2. 회사 노트북이라 앱 설치가 안 돼요.
A. 관리자 권한 없이도 웹 채팅은 Chrome·Edge만 있으면 사용 가능합니다. 4부의 바탕화면 바로가기로 앱처럼 쓰세요. 앱 설치가 꼭 필요하면 경영지원팀 또는 김대표에게 요청.
Q3. 모바일에서도 쓸 수 있나요?
A. 가능합니다. iPhone·Android 앱스토어에서 'Claude' 검색·설치. 같은 계정으로 로그인하면 PC와 내용 공유됩니다. 다만 Cowork는 PC에서만, 모바일은 채팅만 가능.
Q3-1. Google Drive 파일을 Cowork에서 쓰려면 어떻게 하나요?
A. Google Drive for Desktop을 '미러링' 모드로 설치하고, 필요한 폴더만 선택해서 내려받아야 합니다. 전체 Drive를 미러링하면 PC 용량이 부족해집니다. 자세한 설정은 8.2.1 참조. 스트리밍 모드에서는 Cowork가 제대로 작동하지 않으니 반드시 미러링으로 전환하세요.
비용·계정
Q4. Claude는 무료인가요?
A. 무료 계정도 있지만 사용량 제한이 있습니다. 실무 활용엔 Pro 또는 Team 플랜이 적합합니다. 팜보스 그룹은 업무용 계정을 제공하니 경영지원팀 확인.
Q5. 개인 계정과 회사 계정을 같이 쓰면?
A. 가능하지만 분리 권장. 업무 내용은 회사 계정에, 개인 용도는 개인 계정에. 메모리도 각각 따로 관리됩니다.
보안·정보
Q6. Claude에 입력한 내용이 외부로 새나가나요?
A. Anthropic은 사용자 데이터를 모델 학습에 쓰지 않는다고 공식 발표했습니다. 다만 어떤 AI 서비스든 민감 정보는 조심하는 게 원칙입니다. 주민번호·계좌·비밀번호는 절대 입력 금지.
Q7. 고객 개인정보를 Claude에 넣어도 되나요?
A. 원칙적으로 안 됩니다. 분석이 꼭 필요하면 이름은 '고객A·B·C'로 바꾸고, 연락처·주소는 삭제한 뒤 입력. 개인정보 보호법 위반 방지.
Q8. 회의 녹취록을 올려서 요약받아도 되나요?
A. 내부 회의는 가능. 외부 파트너·고객이 포함된 녹취는 당사자 동의 없이 올리지 마세요. 동의 받았어도 민감 내용은 마스킹.
품질·정확성
Q9. Claude가 틀린 정보를 주면?
A. AI는 100% 정확하지 않습니다. 특히 구체적 숫자·날짜·법률·최신 뉴스는 반드시 재확인. Claude의 답을 그대로 공식 문서에 옮기지 말고 항상 검증.
Q10. Claude가 우리 회사 일을 제대로 이해 못 해요.
A. 지침(Preferences) 설정이 부족할 가능성이 높습니다. 5부를 다시 보고 본인 업무 맥락을 더 자세히 입력하세요. 메모리 기능도 활성화. 1~2주 사용하면 Claude가 우리 상황에 훨씬 익숙해집니다.
활용·습관
Q11. 어떤 업무부터 Claude를 써야 하나요?
A. 반복되고 귀찮은 것부터. ① 이메일 답장 → ② 주간 보고 ③ 회의록 정리 순으로 시작하면 효과 가장 큽니다. 창의적 판단·인간 관계 업무는 나중에.
Q12. 습관이 안 들어요. 결국 수동으로 하게 돼요.
A. 바탕화면 바로가기로 '3초 내 접근'을 만드세요(4부). 업무 시작 전 1분만 Claude에 물어보는 습관을 1주일만 붙이면 자리잡습니다.
Q13. 팀원마다 사용법이 달라 혼란스러워요.
A. 이 매뉴얼이 표준입니다. 팀 내에서 5부의 '표준 지침'을 동일하게 세팅하고, 10부의 '프롬프트 템플릿'을 공유·통일하세요. 경영지원팀에 팀별 템플릿 라이브러리 요청 가능.
