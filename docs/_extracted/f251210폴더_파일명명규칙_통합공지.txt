전사 폴더·파일 명명 규칙 및 데이터 표준 심화 가이드(직원 공지용)
1️⃣ 명명 규칙의 핵심 목적
명명 규칙은 단순한 파일 이름 통일이 아니라 회사 전체의 데이터, 정산, AI 자동화 시스템의 기반 구조입니다.
• 문서 검색, 정산, 인수인계가 자동으로 정렬되도록 만드는 표준입니다.
• 파일명만 보고 회사·날짜·버전·업무 내용을 즉시 파악할 수 있어야 합니다.
• 향후 지주회사 체계 확장과 AI 자동 정산 시스템 구축을 위한 주요 기반입니다.
2️⃣ 파일명 기본 구조
[회사코드][연도][월][일][차수]-[업무명]

예:  j25060301-거래처정산
     f25060302-매입현황
3️⃣ 항목별 상세 규칙
① 회사코드
j = 정다운 영농조합
f = 팜보스 농업회사법인
s = 시경 주식회사
fj = 팜보스–정다운 공동 업무

회사코드는 파일을 열기 전에 어느 법인 문서인지 즉시 식별하는 기능을 합니다.
② 연도(YY)
2025년 → 25, 2026년 → 26
두 자리 표기 필수.
정렬성과 데이터베이스 호환을 위한 규칙입니다.
③ 월(MM)
1월 → 01, 9월 → 09
항상 두 자리 표기.
정렬 시 1, 10, 11 순서가 섞이는 문제를 방지.
④ 일(DD)
3일 → 03, 12일 → 12
두 자리 고정.
⑤ 차수
같은 날짜에 여러 파일이 있을 경우 순번 부여.
01 = 첫 번째 파일, 02 = 두 번째 파일.
파일 버전 관리와 AI의 최신 파일 인식에 활용됩니다.
⑥ 업무명
예: 거래처정산, 매입현황, 판매리스트, 재고정리
띄어쓰기 없이 작성.
업무의 목적이 명확히 드러나게 작성합니다.
4️⃣ 폴더 구조 규칙
폴더는 연/월/일까지만 사용하고, 차수는 파일명에만 반영합니다.

📂 25  → 연도 폴더
    📂 06 → 월 폴더
         📂 03 → 일자 폴더
             파일명: j25060301-거래처정산

이유:
• 폴더가 과도하게 세분화되는 것을 방지
• 날짜 기준으로 모든 문서를 한눈에 확인 가능
• 파일 정렬만으로 업무 흐름 파악 가능
5️⃣ 데이터 입력 표준(A~J열)
A열: (날짜) — 거래 발생일 YYYY-MM-DD 형식
B열: 매입처 — 구매처
C열: 공급처 — 실제 물량 공급 주체
D열: 상세(구분/품목) — 품목 카테고리
E열: 품명 — 상품명/판매명(중량·등급·과수 포함 가능)
F열: 중량 — kg, box 등 단위
G열: 등급(과수) — 등급 + 입수
H열: 수량 — BOX 또는 입수
I열: 단가 — 개당 또는 박스 단가
J열: 합계금액 — 자동 계산식

※ J열 이후는 부서별 자유 확장 영역입니다.
6️⃣ 품목(ITEM)과 품명(PRODUCT NAME) 구분
✔ 품목 = 큰 분류 (감, 복숭아, 사과 등) → D열
✔ 품명 = 실제 판매명/상품명 (중량·등급·옵션 포함 가능) → E열

예:
• 품목: 사과
• 품명: 사과 5kg 25과 특대

AI 자동 정산 및 상품 매칭의 Key는 '품명(E열)'입니다.
7️⃣ 품명에 중량·등급·과수가 포함된 경우 규칙
품명(E열)에 중량/등급/과수가 이미 포함된 경우 F열과 G열은 비워도 됩니다.

예시:
• 사과 5kg 25과 특대 → F/G 생략
• 복숭아 백도 4.5kg 상 → F/G 생략
• 샤인머스캣 2송이 프리미엄 → F/G 생략

부분 포함 시:
• 복숭아 백도 4.5kg → F열 생략 / G열에 등급만 작성
8️⃣ 왜 이러한 규칙이 필요한가?
• 부서 간 데이터 연결이 쉬워져 전체 손익 분석이 자동화됩니다.
• 파일 찾기, 정산, 보고 업무 시간이 70% 이상 단축됩니다.
• AI 자동 정산 시스템의 핵심 기반이 됩니다.
• 대표님이 실무에서 점점 벗어나 경영 중심 역할로 전환할 수 있는 구조를 만듭니다.
• 조직 규모가 커져도 혼란 없이 운영 가능한 ‘시스템 경영’이 완성됩니다.
9️⃣ 최종 직원 안내 문구
“전사 모든 파일과 폴더는 회사코드·연월일·차수 기반 명명 규칙을 적용합니다.
폴더는 일자까지, 파일은 차수까지 구분하며,
이 기준은 AI 정산 시스템 구축과 문서 관리 자동화를 위한 핵심 원칙입니다.
A~J열 데이터 입력 기준을 모든 부서가 동일하게 준수 바랍니다.”

🔹 왜 YYYY-MM-DD가 필수인가 

연도 변경 시 데이터 혼선 방지

주·월·연 단위 자동 집계 가능 

외상·통장·재고 자동 매칭 가능 

AI 분석 시 시간축 인식 오류 방지 


 🔹전사 데이터 입력 표준 (A~J열 · 모든 파일 공통)

 모든 매출·매입·외상·재고 파일은 아래 A~J열 구조를 절대 변경하지 않습니다. 







🔹 A~J열 표준 정의 열	항목명	입력 기준

 A	일자	YYYY-MM-DD 형식

 B	매입처	구매처(드롭다운) 

C	매출처	공급처/채널(드롭다운)

 D	상세/구분(품목)	품목 대분류

 E	품명	실제 상품명(중량·등급 포함 가능)

 F	중량	kg / box 등

 G	등급(과수)	등급 + 입수

 H	수량	BOX / EA

 I	단가	개당·박스 단가

 J	합계금액


자동 계산식 ※ J열 이후는 부서별 자유 확장 영역(비고, 담당자 등) 


🔑 전사 공지용 한 문장 요약
모든 파일은 YYYY-MM-DD 기준으로 저장하고,
모든 데이터는 A~J 전사 표준에 맞춰 입력한다.
이 기준은 정산·외상·재고·AI 분석의 공통 언어다.
