# Secrets · 텔레그램 · Pages 셋업 가이드

> 역할: f_trendnewsbot V1 첫 가동·신규 운영자 인계 시 따라가는 단계별 가이드.
> 대상: 운영자(BotFather 토큰 발급 권한 + GitHub repo Settings 권한 보유자).

`docs/canonical/ADR.md` ADR-003 (Pages + 텔레그램 발송) + ADR-004 (Gemini 2.0 Flash LLM provider) 의 실행 단계를 1회 셋업 기준으로 풀어 쓴 문서다. 이 가이드만 따라가면 처음 가동까지 진행할 수 있도록 단계마다 확인 체크리스트를 두었다.

---

## 0. 준비물

- GitHub repo `karajaku/f_trendnewsbot` Settings 권한
- 텔레그램 앱·계정 (BotFather 와 1:1 chat 가능)
- 직원용 단톡방을 만들거나 운영 중인 단톡방 1개 (다이제스트 수신 대상)

---

## 1. 텔레그램 봇 발급

1. 텔레그램에서 `@BotFather` 와 1:1 chat 열기
2. `/newbot` 명령 입력
3. 봇 이름 결정 (예: `farmboss_trend_bot`) — display name 과 username 모두 입력
4. BotFather 가 반환하는 토큰을 복사: `123456:ABC-DEF...` 형태
   - 이 토큰이 곧 `TELEGRAM_BOT_TOKEN` 값이다.
   - **GitHub Secrets 외 다른 곳에 저장 금지** — 평문 메모·노션·sticky note 모두 금지. 유출 시 BotFather 의 `/revoke` 로 즉시 폐기 후 재발급.

---

## 2. 직원 다이제스트 단톡방 생성 + chat_id 획득

1. 텔레그램에서 직원용 단톡방 생성 (또는 기존 단톡방 사용)
2. 위에서 발급한 봇을 단톡방에 초대
   - 봇이 메시지 읽기 권한을 받아야 `getUpdates` 가 동작한다. 단톡방 종류가 "익명 단톡방" 이면 Privacy mode 가 ON 인지 확인 (BotFather `/setprivacy` → Disable).
3. 단톡방에서 봇을 멘션하거나 아무 메시지 1건 발송
4. 브라우저에서 다음 URL 열기 (TOKEN 자리에 ① 의 토큰 그대로 붙임):

   ```
   https://api.telegram.org/bot{TOKEN}/getUpdates
   ```

5. 응답 JSON 의 `result[].message.chat.id` 값을 복사
   - 단톡방은 음수 정수 (예: `-1001234567890`)
   - 1:1 chat 은 양수 정수
   - 이 값이 `TELEGRAM_CHAT_ID` 다.

---

## 3. 운영자 alert chat ID 획득

운영자가 에러 발생 시 짧은 메시지를 받을 목적지다. 직원 단톡방과 분리한다.

- **옵션 A (권장)**: 봇과 1:1 chat — 운영자 1명만 받으면 충분할 때
- **옵션 B**: 운영 전용 별도 단톡방 (운영자 2~3명)

② 와 동일한 방식으로 `chat.id` 추출 → `OPS_ALERT_CHAT_ID` 로 사용한다.

## 4. gh-pages branch 셋업

✅ **이미 완료** — `origin/gh-pages` 가 존재. 미래 운영자용 보존:

```powershell
# (참고용 - 이미 셋업되어 다시 실행할 필요 없음)
git checkout --orphan gh-pages
git rm -rf .
"# Digest archive`nGenerated by f_trendnewsbot." | Out-File -FilePath README.md -Encoding utf8
"User-agent: *`nDisallow: /" | Out-File -FilePath robots.txt -Encoding utf8
git add README.md robots.txt
git commit -m "Initialize gh-pages branch (orphan)"
git push origin gh-pages
git checkout master
```

`dispatchers/pages_publish.py` 가 `git worktree add ... gh-pages` 로 별도 임시 디렉토리에 checkout 한 뒤 `digest/YYYY-MM-DD.html` 을 push 한다 — master 에 영향 0.

## 5. GitHub Pages 활성화

✅ **이미 완료** — `https://karajaku.github.io/f_trendnewsbot/` 가 라이브. 미래 운영자용 보존:

## 6. GitHub Secrets · Variables 등록

`Settings` → 좌측 `Secrets and variables` → **Actions** 탭. Secrets 와 Variables 가 별도 탭이다.

### Gemini API key 발급 (Secrets 등록 전 사전 작업)

ADR-004 (2026-05-19) 에 따라 LLM provider 가 Anthropic Claude 에서 **Google Gemini 2.0 Flash** 로 변경됐다. 무료 tier 영구 운영.

1. https://aistudio.google.com/app/apikey 접속
2. Google 계정 로그인 (운영자 본인 계정)
3. `Create API key` 클릭 → 신규 키 발급 (즉시, 무료)
4. 발급된 키 (`AIzaSy...` 형태) 를 복사 — 이 값이 `GEMINI_API_KEY` 다.

> ⚠ **기존 운영자 마이그레이션**: 이전에 `ANTHROPIC_API_KEY` 를 등록했다면 GitHub Secrets 에서 **삭제 후** 아래 표의 `GEMINI_API_KEY` 를 새로 추가한다. `CLAUDE_MODEL_ID` Variable 도 삭제 후 `GEMINI_MODEL_ID` 로 대체.

### Secrets (암호화 — 한 번 입력 후 다시 못 봄)

| Name | Value 출처 |
| --- | --- |
| `GEMINI_API_KEY` | 위 단계에서 발급한 Gemini API 키 (`AIzaSy...`) |
| `TELEGRAM_BOT_TOKEN` | ① BotFather 에서 받은 토큰 (`123456:ABC-DEF...`) |

### Variables (평문 — 운영자가 언제든 확인 가능)

| Name | Value |
| --- | --- |
| `TELEGRAM_CHAT_ID` | ② 에서 추출한 음수 정수 (예: `-1001234567890`) |
| `OPS_ALERT_CHAT_ID` | ③ 에서 추출한 chat ID |
| `PAGES_BASE_URL` | `https://karajaku.github.io/f_trendnewsbot` (trailing slash 없음) |
| `GEMINI_MODEL_ID` | `gemini-2.0-flash` |

> ⚠ `TELEGRAM_CHAT_ID` 와 `OPS_ALERT_CHAT_ID` 는 **반드시 Variables 에 등록**. Secrets 에 넣으면 workflow 가 `int` 변환 단계에서 마스킹된 `***` 을 받아 실패한다.

## 7. 첫 dry-run 실행

1. Repo 상단 `Actions` 탭
2. 좌측 workflow 목록에서 `daily digest` 선택
3. 오른쪽 `Run workflow` 드롭다운 클릭
4. `dry_run` 체크박스 ✅ 체크 → `Run workflow` 버튼
5. 1~3분 후 운영자 chat (③) 에 메시지 도착

---

## 8. 확인 체크리스트

dry-run 후 다음 항목이 모두 ✅ 면 step7 검증 통과:

- [ ] 운영자 텔레그램 (`OPS_ALERT_CHAT_ID`) 에 메시지 도착
- [ ] 직원 단톡방 (`TELEGRAM_CHAT_ID`) 에는 발송 안 됨 (dry-run 격리 확인)
- [ ] 메시지에 들어 있는 Pages URL 클릭하면 다이제스트 본문 표시
- [ ] Pages 본문 view-source 에 `<meta name="robots" content="noindex,nofollow">` 존재 (AC-2.8)
- [ ] `https://karajaku.github.io/f_trendnewsbot/robots.txt` 가 `User-agent: * / Disallow: /` 표시 (AC-6.4)
- [ ] Actions 로그에 `cron 종료 정상 — 발송 건수=N pages_url=...` 라인
- [ ] Actions run 의 Artifacts 섹션에 `digest-history` 1건 생성

## 9. 트러블슈팅

| 에러 메시지 | 원인 | 해결 |
| --- | --- | --- |
| `ConfigError: missing env vars: [...]` | Secrets/Variables 6 중 일부 누락 | ⑥ 표 다시 확인 |
| `PagesPublishError(stage="write")` | gh-pages branch 미존재 | ④ 셋업 (현재 환경은 완료됨) |
| `PagesPublishError(stage="push")` | `permissions.contents: write` 누락 | `.github/workflows/daily.yml` 의 `permissions` 블록 확인 |
| `PagesPublishError(stage="verify")` | Pages 활성화 안 됨 / 1~2분 전파 지연 | ⑤ 활성화 확인. 60초 timeout 후 재시도 |
| `TelegramSendError("auth")` | `TELEGRAM_BOT_TOKEN` 잘못 또는 봇이 revoke 됨 | ① 재발급 후 Secrets 갱신 |
| `TelegramSendError("bad_request")` | `TELEGRAM_CHAT_ID` 잘못 (또는 봇이 단톡방에 안 초대됨) | ② 음수 정수인지 + 봇 초대 여부 확인 |
| `QuotaExceededError` | 일일 토큰·호출 cap 초과 (`src/summarizer/quota.py`) OR Gemini API rate limit (15 RPM / 1500 RPD) | 정상 동작이지만 운영 검토 — 토큰 사용량 또는 RPM 분석 후 cap 조정 |
| `Gemini ... 400 invalid_request` | `GEMINI_API_KEY` 잘못 또는 만료 | https://aistudio.google.com/app/apikey 에서 재발급 후 GitHub Secrets 갱신 |

cron 자동 실행에서 위 에러가 발생하면 운영자 chat 으로 자동 alert 가 도착한다 (`src/dispatchers/ops_alert.py`, retry 0회 — 무한루프 방지).

## 부록: 로컬 dry-run (선택)

운영자가 본인 노트북에서 워크플로우 없이 직접 1회 실행하려면:

```powershell
# .env 파일 작성 (.env.example 복사 후 실제 값 채움)
Copy-Item .env.example .env
# .env 편집 — Secrets/Variables 6개 값 직접 입력

# 의존성 설치
python -m pip install -e .

# dry-run (운영자 chat 으로만)
python -m src.run_daily --dry-run
```

`.env` 는 `.gitignore` 적용 — 절대 commit 금지.
